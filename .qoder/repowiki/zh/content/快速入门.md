## 简介

sqlmapWebUI 是一个现代化的 SQL 注入测试平台，为安全研究人员提供便捷的 Web 界面来管理 sqlmap 扫描任务。它集成了任务管理、扫描配置、HTTP 请求解析、请求头规则管理以及与 Burp Suite 的扩展集成等核心功能。

本快速入门指南旨在为初学者提供清晰的步骤指导，从环境准备到执行第一个扫描任务，帮助您快速上手使用该系统。

## 环境准备

在开始安装之前，请确保您的系统已安装以下依赖项：

- **Python 3.13+**: 用于运行后端服务。
- **Node.js 20+**: 用于构建和运行前端开发服务器。
- **pnpm 9+**: 前端包管理器，用于安装前端依赖。
- **Java 17+ (用于 Burp Montoya API) 或 Java 11+ (用于 Legacy API)**: 用于构建和运行 Burp Suite 插件。

## 安装与配置

### 后端安装

1.  打开终端，进入后端目录：
    ```bash
    cd src/backEnd
    ```
2.  使用 `uv` 包管理器安装所有依赖，包括第三方库：
    ```bash
    uv sync --extra thirdparty
    ```
3.  （可选）配置环境变量。复制 `.env.example` 文件并重命名为 `.env`，然后根据需要修改配置，如数据库路径、日志级别等。
    ```bash
    cp .env.example .env
    ```

### Burp Suite 插件构建

1.  进入插件目录，根据您的 Burp Suite 版本选择对应的构建方式：
    - **对于 Burp Suite 2023.1+ (推荐使用 Montoya API)**:

2.  构建成功后，JAR 文件将生成在 `target/` 目录下。

## 基本使用

### 创建扫描任务

1.  在浏览器中访问 `http://localhost:5173`。
2.  在任务列表页点击「新建任务」按钮。
3.  在「报文输入」区域，粘贴从浏览器开发者工具（DevTools）中复制的 HTTP 请求。支持的格式包括 cURL (Bash/CMD)、PowerShell、JavaScript fetch 和原始 HTTP 报文。
4.  点击「解析转换」按钮，系统会自动检测格式并将其转换为标准的 HTTP 报文。
5.  （可选）在「HTTP报文编辑」区域，您可以直接编辑转换后的报文。使用 `*` 标记注入点。
6.  在右侧的「扫描配置」区域，选择或配置扫描参数。
7.  配置完成后，点击「提交扫描任务」按钮。

### 使用 HTTP 请求解析器

前端内置了强大的 HTTP 请求解析器，支持多种格式的自动转换。

- **智能格式检测**: 系统会自动识别输入的请求格式（cURL, PowerShell, fetch, 原始 HTTP）。
- **代码编辑器**: 提供行号显示、语法高亮和搜索过滤功能，方便编辑和查看请求。
- **格式支持**:
    - **cURL**: 支持 Bash 和 Windows CMD 格式。
    - **PowerShell**: 支持 `Invoke-WebRequest` 命令。
    - **fetch**: 支持 JavaScript 和 Node.js 的 fetch API 调用。
    - **原始 HTTP 报文**: 支持标准的 HTTP 请求格式。

### 查看任务与结果

- **任务列表**: 在首页或任务列表页，您可以查看所有扫描任务的状态、进度和统计信息。支持多维度过滤（URL/报文关键字、状态、日期范围）和多字段排序。
- **任务详情**: 点击任务列表中的任务，可以进入详情页，查看基础信息、HTTP 请求、扫描配置、扫描结果和实时日志。
- **实时日志**: 在任务详情页的「任务日志」标签页，可以实时查看扫描过程中的日志输出。

## 扫描配置与参数

系统提供了灵活的扫描配置管理功能。

### 扫描配置预设

- **默认配置**: 设置全局默认的扫描参数。
- **常用配置**: 保存常用的配置组合，支持增删改查（CRUD）。
- **历史配置**: 查看历史扫描任务中使用过的配置。

### 关键扫描参数

在创建任务时，您可以配置以下关键参数：

- **检测等级 (Level)**: 从 1 到 5，数字越大，测试的参数越多。
- **风险等级 (Risk)**: 从 1 到 3，数字越大，使用的 Payload 风险越高。
- **目标数据库 (DBMS)**: 指定目标数据库类型，如 MySQL, PostgreSQL, Oracle 等，留空则自动检测。
- **注入技术 (Technique)**: 选择使用的注入技术，如布尔盲注(B)、报错注入(E)、联合查询(U)、时间盲注(T)等。
- **线程数 (Threads)**: 控制并发线程数，以平衡速度和稳定性。

## 常见问题与故障排除

### 后端服务无法启动

- **问题**: 执行 `uv run python main.py` 时出现 Python 模块导入错误。
- **解决方案**: 确保已正确执行 `uv sync --extra thirdparty` 命令安装了所有依赖。检查 Python 版本是否为 3.13 或更高。

### 前端无法连接后端

- **问题**: 前端页面提示无法连接到后端 API。
- **解决方案**: 
    1.  确认后端服务已在 `http://localhost:8775` 成功启动。
    2.  检查前端 `.env.development` 文件中的 `VITE_API_BASE_URL` 是否正确指向后端地址。
    3.  检查后端的跨域配置（CORS），确保允许前端开发服务器的来源。

### Burp Suite 插件加载失败

- **问题**: 在 Burp Suite 中加载 JAR 文件时失败。
- **解决方案**: 检查您的 Java 版本是否满足要求。Montoya API 需要 Java 17+，Legacy API 需要 Java 11+。

### HTTP 请求解析失败

- **问题**: 粘贴请求后，点击「解析转换」按钮，提示“无法识别的请求格式”。
- **解决方案**: 
    1.  确保粘贴的请求是完整的，且格式正确。
    2.  尝试使用最常用的 cURL 格式。
    3.  检查请求中是否有特殊字符或编码问题。
